## 1. 시각적 위계

| 현재 문제 | 개선 방향 |
|---------|---------|
| 제목이 제목처럼 안 보임 — 크기/디자인 부족 | 제목 크기 키우고 시각적 구분 강화 |
| 중요한/중요하지 않은 부분의 색차이 없음 | 제목·소제목·본문·메타 간 명확한 위계 |
| 소제목(이슈 사항, 개선 사항 등) 눈에 안 띔 | 소제목 색상·크기를 더 눈에 띄게 |
| 본문 색이 너무 흐릿함 | 본문은 검은색으로 통일 |
| 구분선 색이 너무 옅음 | 구분선 더 진하게 |

## 2. 간격/여백

| 현재 문제 | 개선 방향 |
|---------|---------|
| 전체적으로 간격 너무 큼 | 섹션 간·항목 간 여백 축소 |
| 행간 너무 넓음 | 행간 타이트하게 조정 |
| 들여쓰기 너무 깊음 | 들여쓰기 깊이 줄이기 |
| 회사 전환 구간이 명확하지 않음 | 회사 바뀌는 곳 확실히 끊어지게 |
| 새 회사가 페이지 중간에서 시작 | 여백 조절해서 페이지 초입에 배치 |

## 3. Bold 사용

| 규칙 | 근거 |
|------|------|
| **개선 사항에서 Bold 전부 제거** | "PPT에 통하는 것, 절대 금지" |
| **적용 결과에서만 성과 Bold** | 어필할 부분만 강조해야 효과 있음 |
| 동사에 Bold 금지 ("구현하여", "도입하고") | Bold 남발하면 강조 효과 소멸 |
| 핵심 수치/성과만 Bold | 예: "50% 단축", "UI 프리징 해소" |

### Good 사례 (리뷰어가 체크한 것)
- "React 도입을 **건의**하고" → 주도성이 드러나는 표현
- "4년간 안정적으로 **확장 가능한 프론트엔드 기반** 확보"

### Bad 사례
- 개선 사항 전체에 Bold → 강조 효과 없음
- 상황 설명에 Bold → 읽는 사람 입장에서 불필요

## 4. 문체

| 규칙 | 설명 |
|------|------|
| 어미 하나만 선택해서 통일 | 현재 "입니다/합니다/이다/하다/있다/한다" 혼재 |
| 이력서와 경력기술서 동일 체계 | 문서 간 톤 일관성 |

## 5. 내용 구조

| 규칙 | 근거 |
|------|------|
| **상황 설명 NO, 내가 한 것 어필** | 상황 설명은 면접에서 할 일 |
| 제목 체계: 회사 → 이슈 → 세부 | 나의 스킬/경험 중심으로 재구성 |
| 중요한 것을 문두에 배치 | 첫 줄에서 핵심이 보여야 함 |
| 들여쓰기 할 건지 말 건지 통일 | 혼재하면 산만해 보임 |

## 6. 오픈소스 섹션

| 현재 | 개선 |
|------|------|
| 단순 나열식 (4-1, 4-2, 4-3) | 5개 정도로 명확한 제목 재설정 |
| "이 부분도 Bold 5개 제목 설정 다시" | 카테고리별 의미 있는 제목 |
